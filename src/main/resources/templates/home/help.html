Ayuda y manual de uso

Manual de uso de la aplicación

Esta aplicación permite crear proyectos de estimación de software y calcular su tamaño, esfuerzo y coste.

El flujo general consiste en crear un proyecto, completar un análisis de tamaño y convertir ese tamaño en esfuerzo.

Acceso, registro e inicio de sesión
Registro público

Desde la pantalla de registro se puede crear una nueva cuenta.

Las cuentas creadas mediante registro público se crean como jefes de proyecto.

Los trabajadores no se registran públicamente, sino que son creados por un administrador o jefe de proyecto.

Inicio de sesión

Para trabajar con proyectos es necesario iniciar sesión con un usuario válido.

Flujo general de uso
  1. Iniciar sesión o registrarse.
  2. Crear un proyecto de estimación.
  3. Configurar una tarifa horaria si se desea calcular coste.
  4. Crear un análisis de tamaño: Puntos Función o Use Case Points.
  5. Completar los datos del análisis.
  6. Revisar el tamaño calculado.
  7. Crear una conversión a esfuerzo mediante Delphi o función de transformación.
  8. Revisar el esfuerzo y el coste estimado.
  9. Generar informes PDF o exportar XML si es necesario.
Tipos de usuario
Administrador

El administrador gestiona usuarios del sistema.

Jefe de proyecto

El jefe de proyecto crea proyectos, trabajadores y asigna permisos.

Trabajador de proyecto

El trabajador accede a proyectos asignados según sus permisos.

Gestión de usuarios
Crear un jefe de proyecto
  1. Entrar con usuario administrador.
  2. Ir al menú Usuarios.
  3. Pulsar Crear usuario.
  4. Indicar nombre de usuario, correo electrónico y contraseña.
  5. Seleccionar el rol Jefe de proyecto.
  6. Guardar.
Crear un trabajador

Un trabajador puede ser creado por un administrador o por un jefe de proyecto.

  1. Ir al menú Usuarios.
  2. Pulsar Crear usuario.
  3. Indicar nombre de usuario, correo electrónico y contraseña.
  4. Si entra un administrador, seleccionar el jefe asociado.
  5. Guardar.
Editar usuarios

Desde la lista de usuarios se pueden editar datos básicos.

Eliminar usuarios

La eliminación solo está disponible cuando no hay información asociada que lo impida.

Mi cuenta

Cada usuario puede consultar sus datos desde Mi cuenta.

Desde esta sección se puede modificar el correo electrónico y cambiar la contraseña.

El nombre de usuario, rol y jefe asociado no se modifican desde la cuenta personal.

Proyectos de estimación

Un proyecto representa el sistema software que se quiere estimar.

Crear un proyecto
  1. Ir al menú Proyectos.
  2. Pulsar Crear proyecto.
  3. Introducir nombre y descripción.
  4. Indicar una tarifa por hora si se desea calcular coste.
  5. Indicar la moneda, por ejemplo EUR.
  6. Guardar.

La tarifa horaria es opcional.

Trabajadores y permisos en proyectos

Un jefe de proyecto puede asignar trabajadores a sus proyectos.

  1. Abrir el detalle del proyecto.
  2. Pulsar Gestionar trabajadores.
  3. Seleccionar el trabajador.
  4. Marcar los permisos del trabajador.
  5. Guardar la asignación.
Permiso para editar datos de estimación

Permite modificar datos internos de los análisis.

Permiso para gestionar conversiones a esfuerzo

Permite crear y gestionar Delphi y funciones de transformación.

Análisis de Puntos Función

El análisis de Puntos Función estima el tamaño funcional del proyecto.

Cómo completar un análisis PF
  1. Abrir el detalle del proyecto.
  2. Crear o abrir el análisis de Puntos Función.
  3. Definir el límite del sistema.
  4. Crear módulos funcionales.
  5. Añadir requisitos de usuario dentro de cada módulo.
  6. Identificar funciones de datos y transaccionales.
  7. Valorar las características generales del sistema.
  8. Revisar o editar la matriz de pesos.
  9. Consultar los resultados.
Módulos PF

Los módulos permiten dividir el sistema en partes funcionales.

Requisitos de usuario

Los requisitos describen necesidades funcionales del sistema.

Funciones de datos

Las funciones de datos representan grupos lógicos de información.

Tipo Significado
ILF Información mantenida por la aplicación.
EIF Información usada por la aplicación pero mantenida externamente.
Funciones transaccionales

Las funciones transaccionales representan operaciones realizadas por el sistema.

Tipo Significado
EI Entrada externa.
EO Salida externa.
EQ Consulta externa.
Complejidad

Cada función se clasifica como simple, media o compleja.

Características generales del sistema

Se valoran de 0 a 5 para ajustar el resultado.

Matriz de pesos

Indica el peso de cada función según tipo y complejidad.

Resultados PF
  • UFP.
  • TDI.
  • VAF.
  • AFP.
Análisis Use Case Points

El análisis Use Case Points estima el tamaño a partir de actores y casos de uso.

Cómo completar un análisis UCP
  1. Abrir el detalle del proyecto.
  2. Crear o abrir el análisis Use Case Points.
  3. Definir el límite del sistema.
  4. Crear actores.
  5. Crear módulos.
  6. Crear casos de uso dentro de los módulos.
  7. Asociar actores a los casos de uso.
  8. Revisar factores técnicos y ambientales.
  9. Consultar los resultados.
Actores

Los actores representan usuarios, sistemas externos u otras entidades.

Complejidad Significado
Simple Interacción sencilla.
Medio Interacción intermedia.
Complejo Interacción más completa.

No se puede eliminar un actor asociado a casos de uso.

Módulos UCP

Los módulos permiten organizar casos de uso.

Casos de uso

Los casos de uso describen funcionalidades del sistema.

Pueden incluir nombre, descripción, actores, condiciones, flujos y número de transacciones.

Complejidad de casos de uso
  • 1 a 3 transacciones: simple.
  • 4 a 7 transacciones: medio.
  • 8 o más transacciones: complejo.
Factores técnicos y ambientales

Los factores permiten ajustar el resultado.

Resultados UCP
  • UAW.
  • UUCW.
  • TCF.
  • ECF.
  • UCP.
Conversión a esfuerzo

Después de calcular el tamaño, se puede convertir en esfuerzo.

Delphi

Delphi usa estimaciones de expertos sobre módulos de referencia.

  1. Abrir un análisis PF o UCP.
  2. Pulsar Iniciar conversión Delphi.
  3. Indicar iteraciones, desviación y expertos.
  4. Registrar estimaciones de expertos.
  5. Revisar el esfuerzo final.
Funciones de transformación

Convierten tamaño en esfuerzo mediante una fórmula lineal.

Esfuerzo = a + b × tamaño

Coste estimado

Si el proyecto tiene tarifa horaria, se calcula el coste.

Coste estimado = esfuerzo estimado × tarifa horaria

XML e informes
Importar y exportar XML

La aplicación permite exportar e importar análisis PF y UCP.

Solo se puede importar si el proyecto no tiene ya un análisis del mismo tipo.

Informes PDF

Los informes PDF permiten guardar o compartir los resultados.

Recomendaciones de uso
  • Crear primero la estructura general del proyecto.
  • Usar módulos para dividir el sistema.
  • Revisar el análisis de tamaño antes de crear conversiones.
  • Configurar tarifa horaria para obtener coste.
  • Generar informes PDF para documentar estimaciones.
  • Exportar XML para conservar datos del análisis.
